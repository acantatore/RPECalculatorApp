## Vision

### 10x Check
Open the app at the start of a training block, enter your current 1RM, select a peaking
template (e.g. "Sheiko, 12 weeks"), and the app automatically generates your entire block
with RPE targets, warmup sets, and E1RM progression targets week by week. Every session
shows up as "today's plan" with one tap to mark complete. That's the 10x product.

## Accepted Scope

- Persist selected palette name to DataStore; restore on cold start with fallback to `Palettes[0]`
- Lift selector: Squat, Bench, Deadlift, OHP, Other — shown in main screen Have card
- Session log: save date, lift, weight, reps, RPE, E1RM to Room database
- Log button appears after E1RM is calculated; confirms success with snackbar

## Implementation Notes

### Room DB schema
```kotlin
@Entity(tableName = "sessions", indices = [Index(value = ["lift", "date"])])
data class SessionEntity(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val date: Long,       // epoch milliseconds
    val lift: String,     // "Squat" | "Bench" | "Deadlift" | "OHP" | "Other"
    val weight: Double,   // in kg always
    val reps: Int,
    val rpe: Double,
    val e1rm: Double
)
```

### Theme persistence
- Add `paletteName: String` to `UserPreferencesData` with default `"Original Purple"`
- `onPaletteChange` saves name to DataStore; cold start restores by name with `Palettes[0]` fallback

### Log button debouncing
- Disable button on tap, re-enable after coroutine completes (success or failure)
- Prevents duplicate rows from rapid taps

### Navigation note
Currently a single `showSettings: Boolean` in `MainActivity`. Adding History will require
either a second boolean or a `sealed class Screen` enum. Design for the enum now even if
History ships later — it's a two-way door and costs nothing.
